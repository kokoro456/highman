🔀 아키텍처 접근 방식 비교

A. 모놀리식 우선

단일 Next.js 앱 + 역할 기반 분리
  • Next.js 풀스택 (API Routes)
  • PostgreSQL (단일 DB)
  • React Native (모바일)
  • Prisma ORM
  • 역할별 라우팅 분리

장점

  • 가장 빠른 MVP 출시
  • 단일 코드베이스 관리 용이
  • 초기 인프라 비용 최소

단점

  • 규모 커지면 분리 필요
  • B2B/B2C 결합도 높음
  • 팀 확장 시 병목

B. 모듈러 모노레포

Turborepo + 도메인 분리 + 공유 패키지
  • Turborepo 모노레포
  • Next.js (B2B 웹 / B2C 웹)
  • NestJS API Gateway
  • React Native (모바일 앱)
  • PostgreSQL + Redis
  • 공유 패키지 (UI, Types, Utils)

장점

  • 코드 재사용 극대화
  • 독립적 배포 가능
  • 팀 확장에 유연
  • 점진적 마이크로서비스 전환 가능

단점

  • 초기 설정 복잡도 높음
  • 빌드 파이프라인 설계 필요

C. 마이크로서비스

서비스별 독립 배포 + API Gateway
  • 서비스별 독립 레포
  • API Gateway (Kong/AWS)
  • 서비스당 개별 DB
  • 메시지 큐 (RabbitMQ)
  • Docker + K8s

장점

  • 완전한 서비스 독립성
  • 무한 확장 가능
  • 기술 스택 자유도

단점

  • 초기 비용 & 복잡도 매우 높음
  • 소규모 팀에 오버엔지니어링
  • 운영 부담 큼
