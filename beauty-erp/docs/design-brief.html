7. 주요 사용 시나리오

시나리오 1: 원장님의 하루

  1. 아침에 대시보드에서 오늘 예약 확인
  2. 고객 방문 → 예약 상태 변경 (확정→진행중→완료)
  3. 시술 후 결제 처리 (카드/현금/회원권)
  4. 퇴근 전 일일 매출 확인

시나리오 2: 고객의 예약

  1. 카카오톡/네이버로 매장 B2C 예약 페이지 접속
  2. 원하는 서비스 선택 → 날짜/시간 선택
  3. 이름/전화번호 입력 → 예약 완료
  4. 카카오 알림톡으로 예약 확인 수신

시나리오 3: 월말 정산

  1. 정산 관리 페이지에서 월별 직원 성과 확인
  2. 인센티브 자동 계산 결과 검토
  3. 엑셀 내보내기로 급여 데이터 다운로드

8. 특별 요청사항

  1. 예약 캘린더가 가장 중요한 화면 — 직원별 컬럼, 시간대별 블록이 한눈에 보여야 함
  2. 고객 VIP 등급 시각적으로 잘 구분 (일반/실버/골드/VIP/VVIP)
  3. 다크모드 기본 지원 — 밤에 매장 사용 시 눈 피로 감소
  4. B2C 예약 페이지는 고객 대상이므로 특히 세련되어야 함
  5. 로딩/빈 상태/에러 상태 각각의 디자인 포함
  6. 토스트 알림 디자인 (성공/에러/정보)

9. 납품 형식

  • 디자인 시스템 (색상 팔레트, 타이포그래피 스케일, 간격, 컴포넌트)
  • 데스크탑 (1280px+) + 모바일 (375px) 두 가지 뷰포트
  • 다크모드 포함
  • 개발자 핸드오프용 스펙
